= GPU Resource Management on OpenShift

Welcome to the GPU Resource Management workshop! In this hands-on session, you'll learn how to simulate GPU hardware, manage GPU reservations through a booking interface, and control workload scheduling with Kueue -- all on OpenShift with Red Hat OpenShift AI.

This workshop uses **fake GPUs** -- simulated GPU accelerators running on CPU-only nodes -- so you can learn GPU resource management patterns without needing real GPU hardware. Everything you learn here applies directly to production GPU clusters.

== What you'll learn

In this workshop, you will:

* **Set up fake GPU nodes** -- understand how the fake-gpu-operator simulates NVIDIA GPUs and MIG partitions on CPU-only clusters
* **Use the GPU Booking App** -- reserve GPU time slots through an OpenShift Console plugin with a calendar interface
* **Configure RHOAI Hardware Profiles** -- connect GPU reservations to OpenShift AI workbenches and model serving
* **Understand Kueue scheduling** -- explore ClusterQueues, LocalQueues, Cohorts, and ResourceFlavors for fair GPU sharing
* **Test preemption** -- observe how reserved workloads preempt unreserved ones using Kueue's cohort borrowing model

The workshop environment consists of several integrated components:

|===
| Component | Purpose

| **Kueue**
| Kubernetes-native job queueing system that manages GPU quotas, fair sharing, and preemption

| **Red Hat OpenShift AI**
| ML platform with workbenches, model serving, and hardware profiles linked to Kueue queues

| **GPU Config Plugin**
| OpenShift Console plugin for selecting GPU profiles (A100, H100, H200, B200, GB200, L40S, T4) and MIG configurations

| **Node Feature Discovery**
| Labels nodes with hardware capabilities for GPU scheduling
|===

== GPU resources in this cluster

Your workshop cluster simulates GPU resources using the fake-gpu-operator. The active GPU profile can be changed at any time using the **GPU Config Plugin** in the OpenShift Console (navigate to **GPU Configuration > GPU Profiles**).

The following GPU profiles are available:

[cols="1,2,1,1,1",options="header"]
|===
| Profile | GPU Name | Memory | MIG | Architecture

| a100 | NVIDIA A100-SXM4-40GB | 40 GiB | Yes | Ampere
| h100 | NVIDIA H100 80GB HBM3 | 80 GiB | Yes | Hopper
| h200 | NVIDIA H200 | 141 GiB | Yes | Hopper
| b200 | NVIDIA B200 | 192 GiB | Yes | Blackwell
| gb200 | NVIDIA GB200 NVL | 192 GiB | Yes | Blackwell
| gb300 | NVIDIA GB300 NVL72 | 288 GiB | Yes | Blackwell Ultra
| vera-rubin | NVIDIA Vera Rubin NVL | 288 GiB | No | Rubin
| l40s | NVIDIA L40S | 48 GiB | No | Ada Lovelace
| t4 | NVIDIA T4 | 16 GiB | No | Turing
|===

Each profile provides 8 full GPUs per node. MIG-capable profiles also provide configurable MIG slice partitions (e.g., 1g.18gb, 2g.35gb, 3g.71gb for H200).

* Kueue-based workload scheduling and quota management
